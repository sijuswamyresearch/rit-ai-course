Module 4: Natural Language Processing & Generative AI

From Words to Wisdom: Transformers, LLMs, and the Future of Language AI

A comprehensive 20-hour intensive course on Applied AI and Machine Learning designed for engineering students from all disciplines. Covers mathematical foundations, classical ML, deep learning, NLP with transformers, and deployment with Streamlit. Includes hands-on projects and capstone implementation.

Author

Siju Swamy

machine learning, artificial intelligence, deep learning, natural language processing, transformers, neural networks, python, data science, AI course, RIT Kottayam, engineering education, OBE curriculum

Introduction: The Language Understanding Challenge

What is Natural Language Processing?

Natural Language Processing (NLP) is the field of artificial intelligence focused on enabling computers to understand, interpret, generate, and manipulate human language. It sits at the intersection of computer science, artificial intelligence, and linguistics.

\[\text{NLP} = \text{Computational Linguistics} + \text{Machine Learning} + \text{Language Understanding}\]

Why is NLP Particularly Challenging?

Human language presents unique computational challenges:

  1. Ambiguity: “I saw a man on a hill with a telescope.” (Who has the telescope?)
  2. Context Dependence: “It’s cold in here.” (Statement of fact or request to close the window?)
  3. Compositionality: Meaning emerges from combining words in specific ways
  4. Variability: Same meaning expressed in countless ways
  5. Evolution: Language constantly changes with new words, slang, and usage patterns

The NLP Pipeline

A typical NLP system processes text through several stages:

Raw Text → Tokenization → Representation → Processing → Output
Stage Purpose Example
Tokenization Split text into units “Hello world” → [“Hello”, “world”]
Representation Convert to numerical form Words → Vectors
Processing Apply ML/AI models Classification, generation
Output Produce result Label, translation, answer

Historical Background: The Evolution of NLP

The Rule-Based Era (1950s-1980s)

Early Beginnings

The Georgetown-IBM experiment (1954) automatically translated 60 Russian sentences to English using hand-crafted rules. This sparked optimism but also revealed the complexity of language.

Chomsky’s Formal Grammars (1957)

Noam Chomsky proposed that language has an underlying mathematical structure [1]:

\[\text{Sentence} \rightarrow \text{Noun Phrase} + \text{Verb Phrase}\]

This led to formal grammar approaches, but natural language proved too messy for rigid rules.

The Statistical Revolution (1980s-2000s)

N-gram Models

Instead of rules, use probability:

\[P(w_n | w_1, w_2, \ldots, w_{n-1}) \approx P(w_n | w_{n-1}, w_{n-2}, \ldots, w_{n-k})\]

Limitation: Cannot capture long-range dependencies due to computational constraints.

Hidden Markov Models (HMMs)

For sequence labeling tasks (like part-of-speech tagging):

\[P(X|Y) = \prod_{i=1}^{T} P(x_i | y_i) \cdot P(y_i | y_{i-1})\]

Where \(X\) is the observed sequence (words) and \(Y\) is the hidden sequence (tags).

The Neural Network Era (2010s-Present)

Word Embeddings (2013)

Word2Vec [2] revolutionized NLP by representing words as dense vectors:

\[\text{king} - \text{man} + \text{woman} \approx \text{queen}\]

This showed that semantic relationships could be captured through vector arithmetic.

Recurrent Neural Networks (RNNs)

RNNs process sequences by maintaining a hidden state:

\[h_t = f(W_{xh} x_t + W_{hh} h_{t-1} + b_h)\]

Problem: Vanishing gradients make it hard to learn long-range dependencies.

Long Short-Term Memory (LSTM) [3]

LSTMs introduced gates to control information flow:

\[\text{Forget Gate: } f_t = \sigma(W_f \cdot [h_{t-1}, x_t] + b_f)\] \[\text{Input Gate: } i_t = \sigma(W_i \cdot [h_{t-1}, x_t] + b_i)\] \[\text{Output Gate: } o_t = \sigma(W_o \cdot [h_{t-1}, x_t] + b_o)\]

The Transformer Revolution (2017)

Vaswani et al. [4] introduced the Transformer architecture, which completely abandoned recurrence in favor of self-attention:

\[\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V\]

This enabled: - Parallel processing (much faster training) - Direct modeling of long-range dependencies - The foundation for all modern LLMs

From Classical NLP to Deep Learning NLP

Classical NLP Approach

Classical NLP relied on:

  1. Hand-crafted features: Linguistic rules, dictionaries, patterns
  2. Shallow models: Naive Bayes, SVM, logistic regression
  3. Pipeline architecture: Separate components for each subtask
Text → POS Tagging → Parsing → NER → Semantic Analysis → Output

Example: Spam Detection with Naive Bayes

\[P(\text{spam}|\text{text}) \propto P(\text{spam}) \prod_{i} P(w_i|\text{spam})\]

Deep Learning NLP Approach

Deep learning transforms the pipeline:

  1. Learned representations: Word embeddings, contextual embeddings
  2. End-to-end learning: Single model handles multiple aspects
  3. Transfer learning: Pre-train on large corpus, fine-tune for specific task
Text → Embedding Layer → Neural Network → Output

Key Differences

Aspect Classical NLP Deep Learning NLP
Features Hand-engineered Learned automatically
Data Needs Small (100s-1000s) Large (10000s-millions)
Computation CPU-friendly GPU/TPU required
Interpretability High (rules visible) Low (black box)
Performance Good for simple tasks State-of-the-art
Development Time Weeks-months Hours-days (with pre-trained)

When to Use Each Approach

Classical NLP is still preferred when: - Data is limited (< 1000 examples) - Interpretability is critical (legal, medical) - Computational resources are constrained - Task is well-defined and simple

Deep Learning NLP is preferred when: - Large amounts of data are available - State-of-the-art performance is needed - Task involves complex language understanding - Computational resources are available

Mathematical Foundations of Modern NLP

Word Representations: From One-Hot to Embeddings

One-Hot Encoding (Classical)

Each word is a sparse vector with a 1 at its vocabulary index:

\[\text{cat} = [0, 0, 1, 0, 0, \ldots, 0]\]

Problems: No semantic meaning, high dimensionality, orthogonal vectors.

Word Embeddings (Modern)

Dense vectors where similar words are close in vector space:

\[\text{cat} = [0.23, -0.45, 0.67, \ldots, 0.12] \in \mathbb{R}^{300}\]

Word2Vec Objective (Skip-gram):

\[\max \sum_{(w,c) \in D} \log P(c|w) = \max \sum_{(w,c) \in D} \log \frac{\exp(v_c^T v_w)}{\sum_{c' \in V} \exp(v_{c'}^T v_w)}\]

Contextual Embeddings (BERT, GPT)

Modern embeddings depend on context:

\[\text{bank}_{\text{river}} \neq \text{bank}_{\text{money}}\]

BERT uses bidirectional context:

\[\text{Representation} = \text{Transformer}([\text{CLS}], w_1, w_2, \ldots, w_n, [\text{SEP}])\]

The Attention Mechanism

Scaled Dot-Product Attention

\[\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V\]

Where: - \(Q\) (Query): What we’re looking for - \(K\) (Key): What each position contains - \(V\) (Value): The actual content at each position - \(d_k\): Dimension of keys (scaling prevents softmax saturation)

Multi-Head Attention

\[\text{MultiHead}(Q, K, V) = \text{Concat}(\text{head}_1, \ldots, \text{head}_h)W^O\]

\[\text{head}_i = \text{Attention}(QW_i^Q, KW_i^K, VW_i^V)\]

This allows the model to attend to different representation subspaces simultaneously.

Transformer Architecture

Positional Encoding

Since Transformers have no inherent notion of sequence order, positional information is added:

\[\text{PE}_{(pos, 2i)} = \sin\left(\frac{pos}{10000^{2i/d_{\text{model}}}}\right)\] \[\text{PE}_{(pos, 2i+1)} = \cos\left(\frac{pos}{10000^{2i/d_{\text{model}}}}\right)\]

Full Transformer Block

  1. Multi-Head Self-Attention
  2. Add & Normalize: $ (x + (x)) $
  3. Feed-Forward Network: $ (x) = (0, xW_1 + b_1)W_2 + b_2 $
  4. Add & Normalize

Language Modeling Objective

Autoregressive (GPT-style)

\[P(x_1, \ldots, x_T) = \prod_{t=1}^{T} P(x_t | x_1, \ldots, x_{t-1})\]

Masked Language Modeling (BERT-style)

\[\max \sum_{i=1}^{T} \log P(x_i | x_{\backslash \text{masked}})\]

Large Language Models (LLMs)

GPT-2 1.5B 40GB text 1024

Scaling Laws

Performance improves predictably with scale [5]:

\[L = \left(\frac{L_c}{C}\right)^{\alpha_C} + \left(\frac{L_D}{D}\right)^{\alpha_D} + L_0\]

Where: - \(C\): Compute budget - \(D\): Dataset size - \(L_c, L_D, \alpha_C, \alpha_D, L_0\): Constants

Emergent Abilities

As models scale, they develop capabilities not present in smaller models:

  1. In-context Learning: Learn from examples in the prompt
  2. Chain-of-Thought: Multi-step reasoning
  3. Cross-modal Understanding: Connect text with other modalities
  4. Tool Use: Call APIs, use calculators, write code

Fine-Tuning Approaches

Full Fine-Tuning

Update all parameters on task-specific data:

\[\theta_{\text{new}} = \theta_{\text{pretrained}} - \eta \nabla_\theta \mathcal{L}_{\text{task}}\]

Parameter-Efficient Fine-Tuning (LoRA)

Only update low-rank decomposition matrices:

\[W_{\text{new}} = W_{\text{frozen}} + \Delta W = W_{\text{frozen}} + BA\]

Where \(B \in \mathbb{R}^{d \times r}\), \(A \in \mathbb{R}^{r \times k}\), and \(r \ll \min(d, k)\).

Retrieval-Augmented Generation (RAG)

The Knowledge Problem

LLMs have inherent limitations:

  1. Static Knowledge: Training data has a cutoff date
  2. No Private Data: Cannot access company documents
  3. Hallucination: Confidently generates incorrect information
  4. No Attribution: Cannot cite sources

RAG Architecture

RAG combines retrieval with generation:

\[P(y|x) = \sum_{d \in D} P(y|x, d) \cdot P(d|x)\]

Where: - \(x\): Input query - \(d\): Retrieved document - \(y\): Generated output - \(D\): Document corpus

Vector Search Fundamentals

Embedding Documents

\[e_d = \text{Encoder}(d) \in \mathbb{R}^{768}\]

Similarity Search

\[\text{similarity}(q, d) = \frac{e_q \cdot e_d}{\|e_q\| \|e_d\|}\]

Approximate Nearest Neighbors

For large corpora, use approximate methods: - FAISS: Facebook AI Similarity Search - Annoy: Approximate Nearest Neighbors Oh Yeah - HNSW: Hierarchical Navigable Small World

RAG Implementation Pipeline

# Simplified RAG pipeline
from sentence_transformers import SentenceTransformer
from sklearn.metrics.pairwise import cosine_similarity
import numpy as np

class SimpleRAG:
    def __init__(self, documents, model_name='all-MiniLM-L6-v2'):
        self.model = SentenceTransformer(model_name)
        self.documents = documents
        self.doc_embeddings = self.model.encode(documents)
    
    def retrieve(self, query, top_k=3):
        query_embedding = self.model.encode([query])
        similarities = cosine_similarity(query_embedding, self.doc_embeddings)[0]
        top_indices = np.argsort(similarities)[-top_k:][::-1]
        return [self.documents[i] for i in top_indices]
    
    def generate_answer(self, query, llm):
        context = "\n".join(self.retrieve(query))
        prompt = f"Context: {context}\n\nQuestion: {query}\n\nAnswer:"
        return llm.generate(prompt)

Connections with Classical ML and Deep Learning

NLP as a Special Case of ML

NLP follows the same fundamental ML pipeline:

Data → Features → Model → Evaluation

But with domain-specific considerations:

ML Component Classical ML NLP Adaptation
Data Tabular, numeric Text sequences
Features Hand-crafted Embeddings, tokens
Model Any classifier RNN, Transformer
Evaluation Accuracy, F1 BLEU, ROUGE, perplexity

From CNNs to Transformers

The evolution of architectures shows a trend toward more flexible, attention-based models:

Architecture Primary Use Key Innovation
CNN Images Local feature detection
RNN Sequences Sequential processing
LSTM Long sequences Gated memory
Transformer Any sequence Self-attention
Vision Transformer Images Apply Transformer to patches

Transfer Learning Paradigm

The same transfer learning principles apply across domains:

  1. Pre-train on large, general dataset
  2. Fine-tune on task-specific data
  3. Evaluate on held-out test set

This works for: - Vision: ImageNet → Medical images - Language: Wikipedia → Legal documents - Speech: Librispeech → Accented speech

Basic Implementations

Text Classification with Traditional ML

import numpy as np
from sklearn.feature_extraction.text import TfidfVectorizer
from sklearn.linear_model import LogisticRegression
from sklearn.metrics import classification_report
from sklearn.model_selection import train_test_split

# Sample data
texts = [
    "I love this product! It's amazing!",
    "Terrible experience, would not recommend.",
    "Great quality and fast shipping.",
    "Waste of money, broke after one day.",
    "Excellent customer service.",
    "Poor quality, very disappointed."
]
labels = [1, 0, 1, 0, 1, 0]  # 1=positive, 0=negative

# Traditional pipeline
vectorizer = TfidfVectorizer(max_features=1000, ngram_range=(1, 2))
X = vectorizer.fit_transform(texts)

X_train, X_test, y_train, y_test = train_test_split(X, labels, test_size=0.3, random_state=42)

# Train classifier
clf = LogisticRegression(random_state=42)
clf.fit(X_train, y_train)

# Evaluate
y_pred = clf.predict(X_test)
print(classification_report(y_test, y_pred))

Sentiment Analysis with Transformers

from transformers import pipeline

# Load pre-trained model
classifier = pipeline("sentiment-analysis", model="distilbert-base-uncased-finetuned-sst-2-english")

# Analyze text
texts = [

for text in texts:
    result = classifier(text)[0]
    print(f"Text: {text}")
    print(f"Sentiment: {result['label']} (confidence: {result['score']:.3f})\n")

Building a Simple Transformer from Scratch

import torch
import torch.nn as nn
import torch.nn.functional as F

class SimpleSelfAttention(nn.Module):
    def __init__(self, embed_size, num_heads):
        super().__init__()
        self.num_heads = num_heads
        self.head_dim = embed_size // num_heads
        
        self.query = nn.Linear(embed_size, embed_size)
        self.key = nn.Linear(embed_size, embed_size)
        self.value = nn.Linear(embed_size, embed_size)
        self.out = nn.Linear(embed_size, embed_size)
        
    def forward(self, x, mask=None):
        batch_size, seq_len, embed_size = x.shape
        
        # Linear projections
        Q = self.query(x).view(batch_size, seq_len, self.num_heads, self.head_dim).transpose(1, 2)
        K = self.key(x).view(batch_size, seq_len, self.num_heads, self.head_dim).transpose(1, 2)
        V = self.value(x).view(batch_size, seq_len, self.num_heads, self.head_dim).transpose(1, 2)
        
        # Attention scores
        scores = torch.matmul(Q, K.transpose(-2, -1)) / (self.head_dim ** 0.5)
        
        if mask is not None:
            scores = scores.masked_fill(mask == 0, -1e9)
        
        # Attention weights
        attn = F.softmax(scores, dim=-1)
        
        # Apply attention to values
        out = torch.matmul(attn, V)
        out = out.transpose(1, 2).contiguous().view(batch_size, seq_len, embed_size)
        
        return self.out(out)

# Test the attention mechanism
attention = SimpleSelfAttention(embed_size=64, num_heads=4)
x = torch.randn(2, 10, 64)  # batch_size=2, seq_len=10, embed_size=64
output = attention(x)
print(f"Input shape: {x.shape}")
print(f"Output shape: {output.shape}")

Fine-tuning BERT for Text Classification

from transformers import BertTokenizer, BertForSequenceClassification, Trainer, TrainingArguments
from datasets import load_dataset

# Load dataset
dataset = load_dataset("imdb")

# Load tokenizer and model
tokenizer = BertTokenizer.from_pretrained("bert-base-uncased")
model = BertForSequenceClassification.from_pretrained("bert-base-uncased", num_labels=2)

# Tokenize function
def tokenize_function(examples):
    return tokenizer(examples["text"], padding="max_length", truncation=True, max_length=512)

# Apply tokenization
tokenized_datasets = dataset.map(tokenize_function, batched=True)

# Training arguments
training_args = TrainingArguments(
    output_dir="./results",
    num_train_epochs=3,
    per_device_train_batch_size=16,
    per_device_eval_batch_size=64,
    warmup_steps=500,
    weight_decay=0.01,
    logging_dir="./logs",
    logging_steps=10,
    evaluation_strategy="epoch",
    save_strategy="epoch",
    load_best_model_at_end=True,
)

# Trainer
trainer = Trainer(
    model=model,
    args=training_args,
    train_dataset=tokenized_datasets["train"].shuffle().select(range(1000)),
    eval_dataset=tokenized_datasets["test"].select(range(200)),
)

# Train
trainer.train()

# Evaluate
results = trainer.evaluate()
print(f"Evaluation results: {results}")

Advanced Topics: Glimpses into Modern NLP

1. Multimodal Models

Models that understand both text and other modalities:

CLIP (Contrastive Language-Image Pre-training)

Learns joint image-text representations:

\[\mathcal{L} = -\frac{1}{N} \sum_{i=1}^{N} \log \frac{\exp(\text{sim}(I_i, T_i)/\tau)}{\sum_{j=1}^{N} \exp(\text{sim}(I_i, T_j)/\tau)}\]

DALL-E and Stable Diffusion

Generate images from text descriptions using diffusion models.

2. Code Generation Models

Models specialized for programming:

  • Codex: Fine-tuned GPT for code
  • AlphaCode: Competitive programming
  • Copilot: AI pair programmer

3. Retrieval Methods Beyond Dense Vectors

Sparse Retrieval (BM25)

\[\text{score}(q, d) = \sum_{i=1}^{n} \text{IDF}(q_i) \cdot \frac{f(q_i, d) \cdot (k_1 + 1)}{f(q_i, d) + k_1 \cdot (1 - b + b \cdot \frac{|d|}{\text{avgdl}})}\]

Hybrid Retrieval

Combine dense and sparse methods for better coverage.

4. Prompt Engineering and In-Context Learning

Chain-of-Thought Prompting

Q: Roger has 5 tennis balls. He buys 2 more cans of tennis balls. Each can has 3 tennis balls. How many tennis balls does he have now?

A: Roger started with 5 balls. 2 cans of 3 balls each is 6 balls. 5 + 6 = 11. The answer is 11.

Q: {new_question}
A: Let's think step by step.

ReAct (Reasoning + Acting)

Combine reasoning with tool use:

Thought: I need to find the current weather.
Action: weather_api[location="New York"]
Observation: 72°F, sunny
Thought: Now I can answer.
Answer: The current weather in New York is 72°F and sunny.

5. Efficient Transformers

Linear Attention

Reduce \(O(n^2)\) complexity to \(O(n)\):

\[\text{Attention}(Q, K, V) = \frac{\phi(Q)(\phi(K)^T V)}{\phi(Q)(\phi(K)^T \mathbf{1})}\]

Where \(\phi\) is a feature map (e.g., kernel function).

Mixture of Experts (MoE)

\[\text{MoE}(x) = \sum_{i=1}^{n} G(x)_i \cdot E_i(x)\]

Where \(G\) is a gating network and \(E_i\) are expert networks.

Practical Guidelines

Choosing the Right NLP Approach

Task Data Size Recommended Approach
Simple classification < 1000 Traditional ML (TF-IDF + SVM)
Text classification 1000-10000 Fine-tune small transformer (DistilBERT)
Complex NLU > 10000 Fine-tune large transformer (BERT, RoBERTa)
Generation > 100000 Use API (GPT-4) or fine-tune open-source LLM
RAG system Any Embeddings + vector DB + LLM

Common Pitfalls and Solutions

Problem Cause Solution
Poor performance Insufficient data Data augmentation, transfer learning
Hallucination Model making things up RAG, fact-checking, constrained decoding
Slow inference Large model Quantization, distillation, smaller model
High cost API usage Open-source models, caching, batching
Bias in outputs Training data bias Careful prompt design, filtering, debiasing

Evaluation Metrics

Task Primary Metric Secondary Metrics
Classification Accuracy, F1 Precision, Recall, AUC
Generation BLEU, ROUGE METEOR, BERTScore, human eval
Summarization ROUGE-L BERTScore, faithfulness
Question Answering Exact Match, F1 BLEU, METEOR
Retrieval MRR, NDCG Recall@K, MAP

Summary and Key Takeaways

The NLP Evolution

  1. Rule-based systems (1950s-1980s): Hand-crafted linguistic rules
  2. Statistical methods (1980s-2000s): N-grams, HMMs, probabilistic models
  3. Neural networks (2010s): Word embeddings, RNNs, LSTMs
  4. Transformers (2017-present): Self-attention, pre-training, LLMs

Core Concepts

  1. Word Embeddings: Dense vector representations capturing semantic meaning
  2. Attention Mechanism: Weighted combination of inputs based on relevance
  3. Transformer Architecture: Self-attention + feed-forward + residual connections
  4. Pre-training & Fine-tuning: Transfer learning paradigm for NLP
  5. Large Language Models: Massive transformers with emergent abilities

From Classical to Modern NLP

Aspect Classical Modern
Representation Sparse (one-hot) Dense (embeddings)
Context Limited (n-grams) Full sequence (attention)
Training Task-specific Pre-train + fine-tune
Features Hand-crafted Learned
Performance Good State-of-the-art

The Future of NLP

  1. Multimodal AI: Text + vision + audio understanding
  2. Reasoning: Better logical and mathematical capabilities
  3. Efficiency: Smaller, faster, cheaper models
  4. Reliability: Reduced hallucination, better factuality
  5. Personalization: Models adapted to individual users
