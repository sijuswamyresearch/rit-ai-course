Introduction: The Machine Learning Paradigm

From Traditional Programming to Machine Learning

In traditional programming, we explicitly write rules (the code), provide input data, and receive output answers:

\[\text{Rules} + \text{Data} \rightarrow \text{Answers}\]

Machine Learning fundamentally inverts this paradigm. We provide the data and the corresponding answers (labels), and the algorithm discovers the rules:

\[\text{Data} + \text{Answers} \rightarrow \text{Rules}\]

Function Approximation: The Core Mathematical Framework

Supervised machine learning is fundamentally an exercise in function approximation. We posit the existence of an unknown true function \(f: \mathcal{X} \rightarrow \mathcal{Y}\) that maps input features to output labels:

\[y = f(x) + \epsilon\]

where \(\epsilon\) represents irreducible noise in the data.

Our objective is to construct an approximating function \(\hat{f}(x; \theta)\) parameterized by \(\theta\), such that \(\hat{f}\) minimizes the expected loss:

\[\theta^* = \arg\min_{\theta} \mathbb{E}_{(x,y) \sim \mathcal{D}}[\mathcal{L}(y, \hat{f}(x; \theta))]\]

where \(\mathcal{D}\) is the underlying data distribution and \(\mathcal{L}\) is a loss function.

The Bias-Variance Tradeoff

The expected prediction error can be decomposed into three components:

\[\mathbb{E}[(y - \hat{f}(x))^2] = \underbrace{(\mathbb{E}[\hat{f}(x)] - f(x))^2}_{\text{Bias}^2} + \underbrace{\mathbb{E}[(\hat{f}(x) - \mathbb{E}[\hat{f}(x)])^2]}_{\text{Variance}} + \underbrace{\sigma^2}_{\text{Irreducible Error}}\]

  • Bias: Error from overly simplistic assumptions. High bias leads to underfitting.
  • Variance: Error from sensitivity to small fluctuations in training data. High variance leads to overfitting.
  • Irreducible Error: Noise inherent in the problem that cannot be reduced.

The goal of model selection is to find the optimal balance between bias and variance.

Regression: Predicting Continuous Values

Linear Regression: Theory and Derivation

The Model

Given a dataset \(\{(x^{(i)}, y^{(i)})\}_{i=1}^n\) where \(x^{(i)} \in \mathbb{R}^d\) and \(y^{(i)} \in \mathbb{R}\), linear regression assumes:

\[\hat{y}^{(i)} = \beta_0 + \beta_1 x_1^{(i)} + \beta_2 x_2^{(i)} + \cdots + \beta_d x_d^{(i)} = \beta^T x^{(i)}\]

In matrix notation, with \(X \in \mathbb{R}^{n \times (d+1)}\) (including intercept column) and \(\beta \in \mathbb{R}^{d+1}\):

\[\hat{y} = X\beta\]

Ordinary Least Squares (OLS) Solution

The OLS method minimizes the Residual Sum of Squares (RSS):

\[RSS(\beta) = \sum_{i=1}^{n} (y^{(i)} - \hat{y}^{(i)})^2 = (y - X\beta)^T(y - X\beta)\]

Taking the gradient with respect to \(\beta\) and setting it to zero:

\[\frac{\partial RSS}{\partial \beta} = -2X^T(y - X\beta) = 0\]

Solving for \(\beta\) yields the Normal Equation:

\[\beta^* = (X^TX)^{-1}X^Ty\]

Probabilistic Interpretation

Assuming the errors are independently and identically distributed (i.i.d.) following a Gaussian distribution:

\[\epsilon^{(i)} \sim \mathcal{N}(0, \sigma^2)\]

The likelihood function becomes:

\[L(\beta) = \prod_{i=1}^{n} \frac{1}{\sqrt{2\pi}\sigma} \exp\left(-\frac{(y^{(i)} - \beta^T x^{(i)})^2}{2\sigma^2}\right)\]

Maximizing the log-likelihood is equivalent to minimizing RSS, establishing that OLS is the Maximum Likelihood Estimator (MLE) under Gaussian noise assumptions.

Regularization: Ridge and Lasso

To prevent overfitting, we add penalty terms:

Ridge Regression (L2 Regularization): \[\beta^* = \arg\min_{\beta} \left\{ \|y - X\beta\|^2_2 + \lambda \|\beta\|^2_2 \right\}\]

Solution: \(\beta^* = (X^TX + \lambda I)^{-1}X^Ty\)

Lasso Regression (L1 Regularization): \[\beta^* = \arg\min_{\beta} \left\{ \|y - X\beta\|^2_2 + \lambda \|\beta\|_1 \right\}\]

Lasso has the desirable property of inducing sparsity — it can shrink coefficients exactly to zero, performing automatic feature selection.

import numpy as np
import matplotlib.pyplot as plt
from sklearn.linear_model import LinearRegression, Ridge, Lasso
from sklearn.preprocessing import PolynomialFeatures
from sklearn.pipeline import Pipeline

# Generate synthetic data
np.random.seed(42)
n_samples = 100
X = np.sort(np.random.uniform(0, 10, n_samples)).reshape(-1, 1)
y = 2.5 * X.squeeze() + 10 + np.random.normal(0, 5, n_samples)

# Fit Linear Regression
lr = LinearRegression()
lr.fit(X, y)

# Generate predictions
X_test = np.linspace(0, 10, 200).reshape(-1, 1)
y_pred = lr.predict(X_test)

# Calculate confidence intervals
n_bootstraps = 1000
bootstrap_preds = []
for _ in range(n_bootstraps):
    indices = np.random.choice(n_samples, n_samples, replace=True)
    lr_boot = LinearRegression()
    lr_boot.fit(X[indices], y[indices])
    bootstrap_preds.append(lr_boot.predict(X_test))

bootstrap_preds = np.array(bootstrap_preds)
lower_bound = np.percentile(bootstrap_preds, 2.5, axis=0)
upper_bound = np.percentile(bootstrap_preds, 97.5, axis=0)

# Plotting
plt.figure(figsize=(10, 6))
plt.scatter(X, y, alpha=0.6, label='Data Points', color='steelblue')
plt.plot(X_test, y_pred, 'r-', linewidth=2, label=f'OLS: y = {lr.coef_[0]:.2f}x + {lr.intercept_:.2f}')
plt.fill_between(X_test.squeeze(), lower_bound, upper_bound, alpha=0.2, color='red', label='95% Confidence Interval')
plt.xlabel('Feature X')
plt.ylabel('Target y')
plt.title('Linear Regression with Bootstrap Confidence Intervals')
plt.legend()
plt.grid(True, alpha=0.3)
plt.show()

print(f"R² Score: {lr.score(X, y):.4f}")
print(f"Coefficients: {lr.coef_}")

Polynomial Regression

When relationships are non-linear, we can transform features using polynomial basis functions:

\[\hat{y} = \beta_0 + \beta_1 x + \beta_2 x^2 + \beta_3 x^3 + \cdots + \beta_p x^p\]

This remains a linear model because it’s linear in the parameters \(\beta\).

Show the Python Code 
import numpy as np
import matplotlib.pyplot as plt
from sklearn.preprocessing import PolynomialFeatures
from sklearn.linear_model import LinearRegression
from sklearn.metrics import mean_squared_error

# Generate non-linear data
np.random.seed(42)
n_samples = 50
X = np.sort(np.random.uniform(0, 4, n_samples)).reshape(-1, 1)
y = np.sin(X.squeeze()) + 0.1 * np.random.randn(n_samples)

X_test = np.linspace(0, 4, 200).reshape(-1, 1)

degrees = [1, 3, 5, 10]
fig, axes = plt.subplots(2, 2, figsize=(14, 10))
axes = axes.ravel()

for idx, degree in enumerate(degrees):
    # Create polynomial features
    poly_features = PolynomialFeatures(degree=degree)
    X_poly = poly_features.fit_transform(X)
    X_test_poly = poly_features.transform(X_test)
    
    # Fit model
    model = LinearRegression()
    model.fit(X_poly, y)
    y_pred = model.predict(X_test_poly)
    
    # Calculate MSE
    y_train_pred = model.predict(X_poly)
    mse = mean_squared_error(y, y_train_pred)
    
    axes[idx].scatter(X, y, alpha=0.6, color='steelblue', label='Data')
    axes[idx].plot(X_test, y_pred, 'r-', linewidth=2, label=f'Degree {degree}')
    axes[idx].set_title(f'Polynomial Degree {degree}\nMSE: {mse:.4f}')
    axes[idx].set_xlabel('X')
    axes[idx].set_ylabel('y')
    axes[idx].legend()
    axes[idx].grid(True, alpha=0.3)

plt.tight_layout()
plt.show()

Classification: Predicting Discrete Labels

Logistic Regression: From Linear to Probabilistic

The Sigmoid Function

For binary classification, we need outputs bounded between 0 and 1. The sigmoid (logistic) function provides this:

\[\sigma(z) = \frac{1}{1 + e^{-z}}\]

Key properties: - \(\sigma(z) \in (0, 1)\) for all \(z \in \mathbb{R}\) - \(\sigma(0) = 0.5\) (decision boundary) - Derivative: \(\sigma'(z) = \sigma(z)(1 - \sigma(z))\)

The Logistic Regression Model

\[P(y=1|x) = \sigma(\beta^T x) = \frac{1}{1 + e^{-\beta^T x}}\]

The log-odds (logit) is linear:

\[\log\left(\frac{P(y=1|x)}{1 - P(y=1|x)}\right) = \beta^T x\]

Maximum Likelihood Estimation

The likelihood for binary classification:

\[L(\beta) = \prod_{i=1}^{n} P(y^{(i)}=1|x^{(i)})^{y^{(i)}} (1 - P(y^{(i)}=1|x^{(i)}))^{1-y^{(i)}}\]

The log-likelihood (to maximize):

\[\ell(\beta) = \sum_{i=1}^{n} \left[ y^{(i)} \log(\sigma(\beta^T x^{(i)})) + (1-y^{(i)}) \log(1 - \sigma(\beta^T x^{(i)})) \right]\]

Equivalently, we minimize the Binary Cross-Entropy Loss:

\[\mathcal{L}(\beta) = -\frac{1}{n} \sum_{i=1}^{n} \left[ y^{(i)} \log(\hat{y}^{(i)}) + (1-y^{(i)}) \log(1 - \hat{y}^{(i)}) \right]\]

Gradient Descent for Logistic Regression

The gradient of the cross-entropy loss:

\[\frac{\partial \mathcal{L}}{\partial \beta} = \frac{1}{n} X^T (\hat{y} - y)\]

Update rule: \[\beta := \beta - \alpha \frac{1}{n} X^T (\hat{y} - y)\]

Multinomial Logistic Regression (Softmax)

For multi-class classification with \(K\) classes, we use the softmax function:

\[P(y=k|x) = \frac{e^{\beta_k^T x}}{\sum_{j=1}^{K} e^{\beta_j^T x}}\]

The Cross-Entropy Loss for multi-class:

\[\mathcal{L} = -\frac{1}{n} \sum_{i=1}^{n} \sum_{k=1}^{K} \mathbb{I}(y^{(i)} = k) \log P(y=k|x^{(i)})\]

Tree-Based Models: Non-Parametric Learning

Decision Trees: Recursive Partitioning

The Algorithm

A decision tree recursively partitions the feature space into regions. At each node, it selects the feature \(j\) and threshold \(s\) that minimize the impurity:

For Regression (MSE criterion): \[\min_{j,s} \left[ \min_{c_1} \sum_{x_i \in R_1(j,s)} (y_i - c_1)^2 + \min_{c_2} \sum_{x_i \in R_2(j,s)} (y_i - c_2)^2 \right]\]

where \(R_1(j,s) = \{x | x_j \leq s\}\) and \(R_2(j,s) = \{x | x_j > s\}\).

For Classification:

  1. Gini Impurity: \[G = 1 - \sum_{k=1}^{K} p_k^2\]

  2. Entropy: \[H = -\sum_{k=1}^{K} p_k \log_2(p_k)\]

  3. Information Gain: \[IG = H(\text{parent}) - \sum_{v \in \text{values}} \frac{|S_v|}{|S|} H(S_v)\]

Overfitting and Pruning

Decision trees are prone to overfitting because they can create arbitrarily complex boundaries. Control methods:

  1. Pre-pruning: Limit tree depth, minimum samples per leaf, or maximum leaf nodes.
  2. Post-pruning: Grow full tree, then remove branches that don’t improve validation performance.
Show the Python Code 
import numpy as np

Ensemble Methods: The Wisdom of Crowds

Random Forest: Bagging (Bootstrap Aggregating)

Random Forest builds multiple decorrelated trees and averages their predictions.

Algorithm: 1. For \(b = 1\) to \(B\): - Draw bootstrap sample \(Z^*\) of size \(n\) from training data - Grow tree \(T_b\) on \(Z^*\): - At each node, randomly select \(m\) features (typically \(m = \sqrt{p}\)) - Choose best split among \(m\) features - Output tree \(T_b\)

  1. Prediction:
    • Regression: \(\hat{f}(x) = \frac{1}{B} \sum_{b=1}^{B} T_b(x)\)
    • Classification: \(\hat{f}(x) = \text{majority vote} \{T_b(x)\}_{b=1}^{B}\)

Why it works: By averaging many trees, variance is reduced without increasing bias. The random feature selection decorrelates trees.

Gradient Boosting: Sequential Error Correction

Gradient Boosting builds trees sequentially, where each tree corrects the errors of its predecessors.

Algorithm: 1. Initialize with constant value: \(F_0(x) = \arg\min_\gamma \sum_{i=1}^{n} L(y_i, \gamma)\)

  1. For \(m = 1\) to \(M\):
    • Compute pseudo-residuals: \(r_{im} = -\left[\frac{\partial L(y_i, F(x_i))}{\partial F(x_i)}\right]_{F=F_{m-1}}\)
    • Fit tree \(h_m(x)\) to residuals \(r_{im}\)
    • Compute step size: \(\gamma_m = \arg\min_\gamma \sum_{i=1}^{n} L(y_i, F_{m-1}(x_i) + \gamma h_m(x_i))\)
    • Update: \(F_m(x) = F_{m-1}(x) + \nu \gamma_m h_m(x)\)

where \(\nu\) is the learning rate (shrinkage parameter).

XGBoost: Extreme Gradient Boosting

XGBoost enhances gradient boosting with:

  1. Regularization: Adds L1 and L2 penalties to control model complexity \[\mathcal{L} = \sum_i l(\hat{y}_i, y_i) + \sum_k \Omega(f_k)\] where \(\Omega(f) = \gamma T + \frac{1}{2}\lambda \|w\|^2\)

  2. Second-order Taylor expansion for faster optimization

  3. Handling missing values automatically

  4. Parallel processing for tree construction

                         np.linspace(y_min, y_max, 300))

    axes[idx].contourf(xx, yy, Z, alpha=0.3, cmap='RdBu')

    axes[idx].set_title(f'{name}\nTrain Acc: {model.score(X_train, y_train):.3f}\nTest Acc: {acc:.3f}')

Unsupervised Learning: Discovering Hidden Structure

Clustering: K-Means Algorithm

Mathematical Formulation

Given data \(\{x^{(1)}, x^{(2)}, \ldots, x^{(n)}\}\), K-Means minimizes the Within-Cluster Sum of Squares (WCSS):

\[J = \sum_{k=1}^{K} \sum_{x^{(i)} \in C_k} \|x^{(i)} - \mu_k\|^2\]

where \(\mu_k\) is the centroid of cluster \(C_k\).

The Algorithm (Lloyd’s Algorithm)

  1. Initialize: Randomly select \(K\) centroids \(\mu_1, \mu_2, \ldots, \mu_K\)

  2. Repeat until convergence:

    • Assignment step: Assign each point to nearest centroid \[C_k = \{x^{(i)} : \|x^{(i)} - \mu_k\|^2 \leq \|x^{(i)} - \mu_j\|^2 \ \forall j\}\]
    • Update step: Recompute centroids \[\mu_k = \frac{1}{|C_k|} \sum_{x^{(i)} \in C_k} x^{(i)}\]

Convergence and Limitations

  • K-Means is guaranteed to converge (monotonically decreasing \(J\))
  • May converge to local optimum (run multiple times with different initializations)
  • Assumes spherical clusters of similar size
  • Sensitive to outliers

Choosing K: The Elbow Method

Plot WCSS vs. K and look for the “elbow” point where marginal gain decreases.

Show the Python Code 
import numpy as np
import matplotlib.pyplot as plt
from sklearn.cluster import KMeans
from sklearn.datasets import make_blobs

# Generate clustered data
X, y_true = make_blobs(n_samples=500, centers=4, cluster_std=0.8, random_state=42)

# Part 1: Visualize K-Means iterations
fig, axes = plt.subplots(2, 3, figsize=(18, 12))

# Elbow method
inertias = []
K_range = range(1, 11)
for k in K_range:
    kmeans = KMeans(n_clusters=k, random_state=42, n_init=10)
    kmeans.fit(X)
    inertias.append(kmeans.inertia_)

axes[0, 2].plot(K_range, inertias, 'bo-', linewidth=2)
axes[0, 2].set_xlabel('Number of Clusters (K)')
axes[0, 2].set_ylabel('Inertia (WCSS)')

axes[0, 2].grid(True, alpha=0.3)
axes[0, 2].annotate('Elbow Point', xy=(4, inertias[3]), 
                     xytext=(6, inertias[3] + 200),
                     arrowprops=dict(arrowstyle='->', color='red'))

Dimensionality Reduction: PCA

Mathematical Foundation

Principal Component Analysis finds orthogonal directions of maximum variance.

Given centered data matrix \(X \in \mathbb{R}^{n \times p}\), we seek projection directions \(w\) that maximize variance:

\[\max_w \frac{1}{n} \|Xw\|^2 \quad \text{subject to} \quad \|w\| = 1\]

This is equivalent to finding eigenvectors of the covariance matrix:

\[\Sigma = \frac{1}{n} X^T X\]

\[\Sigma w = \lambda w\]

The eigenvalues \(\lambda_1 \geq \lambda_2 \geq \cdots \geq \lambda_p\) represent the variance explained by each principal component.

SVD Approach

PCA can be computed via Singular Value Decomposition:

\[X = U \Sigma V^T\]

The principal components are the columns of \(V\), and the explained variance is proportional to the singular values squared.

Explained Variance Ratio

\[\text{Explained Variance Ratio}_k = \frac{\lambda_k}{\sum_{j=1}^{p} \lambda_j}\]

t-SNE: Non-Linear Dimensionality Reduction

The Algorithm

t-Distributed Stochastic Neighbor Embedding (t-SNE) preserves local structure by modeling pairwise similarities.

High-dimensional similarities: \[p_{j|i} = \frac{\exp(-\|x_i - x_j\|^2 / 2\sigma_i^2)}{\sum_{k \neq i} \exp(-\|x_i - x_k\|^2 / 2\sigma_i^2)}\]

Low-dimensional similarities (using t-distribution): \[q_{ij} = \frac{(1 + \|y_i - y_j\|^2)^{-1}}{\sum_{k \neq l} (1 + \|y_k - y_l\|^2)^{-1}}\]

Cost function (KL Divergence): \[KL(P\|Q) = \sum_i \sum_j p_{ij} \log \frac{p_{ij}}{q_{ij}}\]

Key Parameters

  • Perplexity: Controls the effective number of neighbors (typical: 5-50)
  • Learning rate: Step size for gradient descent
  • Early exaggeration: Scales probabilities in early iterations to form tighter clusters

Model Evaluation and Validation

Cross-Validation

K-Fold Cross-Validation

Divide data into \(K\) folds. Train on \(K-1\) folds, validate on the remaining fold. Repeat \(K\) times.

\[CV_{(K)} = \frac{1}{K} \sum_{k=1}^{K} \text{MSE}_k\]

Stratified Cross-Validation

For classification, ensure each fold has approximately the same class distribution.

Performance Metrics

Classification Metrics

Metric Formula Use Case
Accuracy \(\frac{TP + TN}{TP + TN + FP + FN}\) Balanced classes
Precision \(\frac{TP}{TP + FP}\) Minimize false positives
Recall \(\frac{TP}{TP + FN}\) Minimize false negatives
F1-Score \(2 \cdot \frac{\text{Precision} \cdot \text{Recall}}{\text{Precision} + \text{Recall}}\) Balance precision/recall
AUC-ROC Area under ROC curve Overall discriminative ability

Regression Metrics

Metric Formula
MSE \(\frac{1}{n}\sum_{i=1}^{n}(y_i - \hat{y}_i)^2\)
RMSE \(\sqrt{\text{MSE}}\)
MAE \(\frac{1}{n}\sum_{i=1}^{n}|y_i - \hat{y}_i|\)
R² \(1 - \frac{\sum(y_i - \hat{y}_i)^2}{\sum(y_i - \bar{y})^2}\)

Summary and Key Takeaways

Algorithm Type Key Strength Key Weakness
Linear Regression Regression Interpretable, fast Assumes linearity
Logistic Regression Classification Probabilistic, interpretable Linear decision boundary
Decision Trees Both Non-linear, interpretable Overfitting
Random Forest Both Robust, accurate Less interpretable
XGBoost Both State-of-the-art accuracy Hyperparameter tuning

Best Practices

  1. Always start with simple models (linear regression, logistic regression) as baselines
  2. Use cross-validation to get reliable performance estimates
  3. Scale your features for algorithms sensitive to feature magnitude (PCA, K-Means, SVM)
  4. Handle missing data appropriately before training
  5. Interpret your models — accuracy isn’t everything; understanding matters
  6. Consider the business context when choosing metrics and models
