2. Course Rationale & Objectives

This intensive 20-hour foundation course is designed to transition engineering students from foundational mathematics to deploying modern Generative AI applications. Structured strictly within the parameters of Outcome-Based Education (OBE), the curriculum emphasizes a rigorous mathematical foundation followed by practical implementation. The objective is to cultivate analytical reasoning and engineering problem-solving capabilities required for contemporary AI engineering roles.

3. Course Outcomes (COs)

Upon successful completion of the course, students will be able to:

  • CO1: Apply foundational principles of linear algebra, calculus, and probability to formulate machine learning models and optimization strategies. (Cognitive Level: Apply)
  • CO2: Implement and evaluate supervised and unsupervised classical machine learning algorithms using standard Python frameworks and structured datasets. (Cognitive Level: Apply/Analyze)
  • CO3: Develop deep learning architectures, analyzing the performance impact of activation functions (such as ELU, ReLU, and SiLU) and loss configurations. (Cognitive Level: Create/Evaluate)
  • CO4: Design and deploy end-to-end generative AI pipelines, including Retrieval-Augmented Generation (RAG) systems, utilizing modern deployment frameworks. (Cognitive Level: Create)

5. Detailed Syllabus Structure

Module 1: Mathematical Foundations for AI (4 Hours)

  • Linear Algebra: Vector spaces, matrix operations, Orthogonality, Eigenvalues/Eigenvectors, Singular Value Decomposition (SVD).
  • Calculus: Partial derivatives, the Chain rule, gradient intuition for backpropagation.
  • Optimization: Gradient Descent mechanics, Stochastic Gradient Descent (SGD), Adam Optimizer introduction.
  • Probability: Bayesian vs. Frequentist probability, Gaussian distributions, basics of hypothesis testing.
  • Hands-on Component: Python implementation of a Gradient Descent algorithm from scratch to visualize optimization paths.

Module 2: Classical Machine Learning & Data Workflows (6 Hours)

  • Data Lifecycle: GitHub fundamentals for collaboration; Kaggle API for data sourcing; Data cleaning and Feature Engineering.
  • Supervised Learning: Regression (Linear, Polynomial) and Classification (Logistic Regression, Random Forest, XGBoost).
  • Unsupervised Learning: K-Means and Hierarchical clustering methodologies.
  • Dimensionality Reduction: Principles of PCA vs. t-SNE for high-dimensional data visualization.
  • Hands-on Component: End-to-end exploratory data analysis and ML pipeline construction using Scikit-Learn on benchmarking datasets.

Module 3: Deep Learning Architectures (4 Hours)

  • Neural Network Architecture: Multi-layer Perceptrons (MLP) topology and forward propagation.
  • Activation & Loss: Comparative analysis of activation functions (ReLU, ELU, SiLU) and their impact on performance metrics like FID scores. Loss functions (MSE, Cross-entropy).
  • Training Dynamics: Overfitting vs. Underfitting mitigation strategies including Dropout and Early Stopping.
  • Frameworks: Building, compiling, and evaluating models utilizing Keras and TensorFlow backends.
  • Hands-on Component: Designing and training a deep neural network for a classification task.

Module 4: Natural Language Processing & Generative AI (4 Hours)

  • NLP Foundations: Evolution from sequential models to Transformers; understanding the Self-Attention mechanism.
  • Large Language Models (LLMs): GPT architecture fundamentals and Prompt Engineering methodologies (Few-shot, Chain-of-Thought).
  • Modern AI Stack: Interfacing with the Hugging Face ecosystem (pre-trained models and pipelines).
  • RAG Systems: Principles of Retrieval-Augmented Generation utilizing NotebookLM methodologies and vector stores.
  • Hands-on Component: Developing a local RAG-based chatbot for technical documentation querying.

Module 5: Capstone Integration & Deployment (2 Hours)

  • Model Deployment: Principles of ModelOps and lightweight web application frameworks for data science.
  • Streamlit Integration: Transforming trained models (ML or GenAI) into interactive web applications.
  • Deliverable Configuration: Structuring and finalizing GitHub repositories with comprehensive README documentation.
  • Hands-on Component: Guided deployment of the final Capstone project for portfolio presentation.

6. Assessment & Evaluation Strategy (OBE Compliant)

Quality Assurance & Alignment

To maintain rigorous quality assurance and alignment with accreditation standards, the evaluation scheme utilizes both formative and summative assessments mapped directly to the Course Outcomes.

  • Continuous Assessment (30% - Mapped to CO1, CO2): Automated quizzes and conceptual checks focusing on the mathematical frameworks and classical algorithm mechanics.
  • Practical Execution (30% - Mapped to CO2, CO3): Evaluation of “Skeleton Notebooks” where foundational logic and framework implementations (e.g., proper application of activation functions and optimizers) are assessed.
  • Capstone Project (40% - Mapped to CO4, PO3, PO5): Summative evaluation of a deployed, interactive Streamlit application. Rubrics will evaluate code modularity, Git commit history, and model accuracy/utility.
